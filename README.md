# Сетевой чат

- [client](./client) - Клиент для подключения к чату
- [server](./server) - Серверная часть чата

## Сервер

1. Серевер с помощью класса Settings получает настройки из файла ресурсов
   settings.properties
2. После чего стартует сервер, который организован в виде цикла ожидающего
   подключения на основной порт
3. Для входящих соединений создаётся поток с выделеным портом подключения
4. Выделенный поток в цикле ожидает входящих подключений
5. При получении сообщения поток вызывает сервер, который в свою очередь
   раздаёт задание разослать полученное сообщение всем установленным
   подключениям
6. При отключении клиента поток удаляется из списка установленных подключений и
   завершает работу

## Клиент

1. Клиент с помощью класса Settings получает настройки из файла ресурсов
   settings.properties
2. После чего запускается два потока клиента - один для чтения другой для
   записи
3. Поток для чтения в цикле ожидает сообщений с сервера и при получени выводит
   их в консоль и файл лога
4. Поток для ввода ожидает введёных сообщений от пользователя, которые отсылает
   на сервер чата
5. При вводе `/exit` потоки отсоединяются от сервера и прекращают работу
